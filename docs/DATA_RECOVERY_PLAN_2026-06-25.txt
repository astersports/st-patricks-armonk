0. TWO HARD TRUTHS (read first — they shape the whole plan)
══════════════════════════════════════════════════════════════════════════
A. SCRAPING THE WEBSITE CANNOT RECOVER THE FORMS. Every form SUBMISSION
   (baptism/marriage/funeral/sponsor, CCD registration + permissions, parish +
   teen-life registration, mass intentions, volunteer signups, newsletter
   subscribers, private prayer requests) is write-only into the Manus MySQL and is
   read back ONLY through admin-gated endpoints. It was NEVER rendered publicly.
   19 of 34 tables are like this. A scrape recovers ZERO of them. They are
   recoverable ONLY from a MySQL dump (or surviving external mirrors — see §3).

B. THE UPLOADED FILES ARE THE MOST FRAGILE — NO FALLBACK. Homily audio, gallery
   photos, admin-uploaded bulletin PDFs, parish documents, and any birth-cert
   uploads live in a FORGE-MANAGED S3 bucket. The bucket name + credentials are NOT
   in the repo; the DB stores only opaque keys + a relative `/manus-storage/<key>`
   proxy path. Those files are reachable ONLY while a live app instance + a valid
   Forge API key still exist (the proxy re-signs the S3 URL on every request). Once
   Manus tears down, NO artifact we hold can bring the files back — not even a full
   DB dump (it has the keys, not the bytes). PULL THE FILES FIRST.

I CANNOT EXECUTE EITHER FROM THIS ENVIRONMENT: the agent egress proxy denies
outbound to stpatrickinarmonk.org / the Manus origin (org network policy — logged
`connect_rejected 403`; policy says report, not route around). The scrape + file
pull must run somewhere with internet egress (your machine, or allowlist the domain
in this environment's network policy, then a fresh session runs the script below).

Live origins (either may answer if anything is still serving):

4. MYSQL EXPORT SPEC (run once the DB is reachable; FK-order-independent — 0 DB FKs)
══════════════════════════════════════════════════════════════════════════
Connection: the app used a single DATABASE_URL (server/db/_connection.ts) → MySQL.
Parse host/port/user/password/db from it.

STEP 1 — P0 PII subset FIRST (irreplaceable; get this before a full dump can time out):
  mysqldump --host=H --port=P --user=U --password=PW --single-transaction --quick \
    --default-character-set=utf8mb4 --set-gtid-purged=OFF DB \
    ccd_permissions baptism_registrations marriage_inquiries funeral_pre_planning \
    sponsor_certificates ccd_registrations teen_life_registrations \
    parish_registrations mass_intentions > stpat_P0_pii.sql

STEP 2 — full dump (master artifact):
  mysqldump --host=H --port=P --user=U --password=PW --single-transaction --quick \
    --routines --triggers --events --default-character-set=utf8mb4 \
    --set-gtid-purged=OFF DB > stpat_full.sql

STEP 3 — per-table CSV for the sensitive tables (analyst-friendly; no FILE priv needed):
  for T in ccd_permissions baptism_registrations marriage_inquiries \
           funeral_pre_planning sponsor_certificates ccd_registrations \
           teen_life_registrations parish_registrations mass_intentions \
           email_subscriptions volunteer_signups volunteer_need_responses; do
    mysql --host=H --port=P --user=U --password=PW --default-character-set=utf8mb4 \
      --batch --raw -e "SELECT * FROM \`$T\`;" DB \
      | sed 's/\t/","/g; s/^/"/; s/$/"/' > "$T.csv"; done

STEP 4 — harvest the S3/blob keys, then pull the files (do while storage is alive):
  mysql --host=H ... -e "SELECT 'bulletin' t,pdfKey k,pdfUrl u FROM bulletins WHERE pdfKey IS NOT NULL
    UNION ALL SELECT 'document',fileKey,fileUrl FROM parish_documents WHERE fileKey IS NOT NULL
    UNION ALL SELECT 'gallery',imageKey,imageUrl FROM gallery_photos WHERE imageKey IS NOT NULL
    UNION ALL SELECT 'homily',audioKey,audioUrl FROM homilies WHERE audioKey IS NOT NULL
    UNION ALL SELECT 'birthcert',NULL,birthCertUrl FROM baptism_registrations WHERE birthCertUrl IS NOT NULL;" \
    DB > blob_keys.tsv
  # then for each url: curl -L "https://<origin><url>" -o <basename>   (while the proxy + Forge key live)

STEP 5 — content tables (lowest priority; also reconstructable from scrape):
  mysqldump ... DB news_posts events important_dates parish_documents gallery_photos \
    homilies holy_days staff_members ccd_events cyo_teams cyo_games \
    volunteer_opportunities volunteer_needs parish_faqs site_settings \
    prayer_intentions saint_streaks > stpat_content.sql

══════════════════════════════════════════════════════════════════════════
5. SCRAPE SPEC (published content + media; run where there's egress to the site)
══════════════════════════════════════════════════════════════════════════
The site is a client-rendered React SPA (wouter + lazy pages; a raw GET returns an
empty shell), so use EITHER the public tRPC API (cleaner) OR Playwright/Chromium.

· tRPC API: base `/api/trpc`, superjson-wrapped, GET + batch enabled. ~19
  publicProcedure.query endpoints expose the parish's own structured data:
  bulletins.listPublished, news.listPublished, events.listUpcoming,
  googleCalendar.{parish,ccd,cyo,all}Events, gallery.{listPublished,albums},
  staff.list, faq.listActive, homilies.list, parishSchedule.{getSchedule,getInfo},
  volunteerNeeds.list, holyDays.upcoming, importantDates.allPublished,
  siteSettings.get, prayerWall.getIntentions.
  GET form: /api/trpc/<proc>?input=%7B%22json%22%3Anull%7D ; response under result.data.json.
· Media: every published PDF/audio/image is at `/manus-storage/<key>` (open proxy,
  no auth, 307→signed S3) — but ONLY while the app + Forge key are alive.
· EXCLUDE: /admin/*, /unsubscribe/:token, /ccd-unsubscribe (auth/token-gated).
· Caps to mind: news.listPublished=20, events.listUpcoming=20+future-only,
  bulletins.listPublished=100. Older items beyond the cap = DB-only.

A runnable script is committed alongside this doc: scripts/recover-stpat.mjs
(saves rendered HTML + extracted JSON + downloaded PDFs/audio/images to ./recovery).
Run it from a machine with internet egress, or after allowlisting the domain in this
environment's network policy.
